# Review Code

Perform a thorough code review of the specified file or recent changes.

## Instructions
1. Read the code carefully
2. Check against the review checklist in the reviewer agent
3. Look for bugs, security issues, and improvements
4. Consider project-specific patterns

## Focus Areas
- Correctness and edge cases
- Security vulnerabilities
- Performance concerns
- Code style consistency
- TypeScript type safety

## Output Format
Provide structured feedback with:
- Summary assessment
- What's working well
- Suggestions for improvement
- Critical issues (if any)
- Questions for clarification
